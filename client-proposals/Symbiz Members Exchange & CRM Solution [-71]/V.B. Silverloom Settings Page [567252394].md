5.2. Silverloom Settings Page

  


Requirements

The Silverloom Settings page, accessed from Advanced | Admin | Silverloom Settings, includes the following sections for various configuration settings: 

  


Contact Settings standard section: 

  * Name Format: Set to "FirstName LastName" 
    * Note: This may be overridden in specific scenarios. 



  


  


Life Goal Worksheet Configurations custom section: 

  * Core Values Instructions (memo; used to display the instructions on the members' Contact records) 
  * Life Vision Instructions (memo; used to display the instructions on the members' Contact records)
  * Annual Goals Instructions (memo; used to display the instructions on the members' Contact records)



  


  


Dashboard Configurations section: 

  * Days for Dashboard News Feed (required; number field; validate that the number is between 1 and 60; defaults to 30; used to set how many days' worth of new items display in the News Feed on the Dashboard)



  


  


Printouts custom section: 

  * Instructions: "Click on Upload attachment button or drag and drop in a PDF to attach it to a memo."
  * Book/Resource Impact Worksheet (memo; used to upload the PDF file for this printout)
  * Confidentiality Agreement (memo; used to upload the PDF file for this printout)
  * Growth Ring Health Report (memo; used to upload the PDF file for this printout)
  * Symbuzz Extra Monthly Start (3rd Thursday of prior month) (read-only date; displays the corresponding date for easy reference)
  * Symbuzz Extra Weekly Start (Sunday) (read-only date; displays the corresponding date for easy reference; as with the prompt on the report, this defaults to 2 Sundays prior to the current date, i.e., on Monday, 1/15/24, this would default to Sunday, 1/7/24)



  


  


Notifications section: 

  * Members Exchange Welcome Email Subject (plain text field; used as the subject for the welcome email)
  * Members Exchange Welcome Email Body (memo; supports expressions; contents of this memo are sent as the welcome email; can include information such as a link to the login page, instructions for logging in, a suggestion to bookmark the login page, etc.)



  


  


Registration Settings section:

  * Default Registration Coordinator (drop list of all Contacts; the Contact selected here is the default selection for the Registration Coordinator on all Events; defaults to blank; warning on Save if blank: "Default Registration Coordinator is blank.")
    * Note that as part of setup (both test and live system), a Contact record with a name such as "Main Office" should be set up, with a phone number and/or email address. In the test system, this is to be done by the tester. In the live system, this is to be done by Symbiz.



  


  


Other Notes: 

  * This screen uses a two-column layout.
