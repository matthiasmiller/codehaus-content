8.13. Wiki Report

  


Requirements

This is a two-pane editable report of the Wiki pages in the Solution. By default it would only show active items.

  


This is accessed from: 

  * Members Menu | Wiki | Wiki (bypasses the filter screen to open the report directly)
  * Admin | Wiki | Wiki (bypasses the filter screen to open the report directly) 



  


Title: Wiki

  


Left Pane: 

Columns: 

  * Category (list of Wiki Categories) 



  


Grouped by: 

  * (All Categories) 
  * (Recent Unread)
  * (Recent)
  * List of categories with hierarchies



  


Sorted by: N/A

  


Right Pane: 

Columns: 

  * Read/Unread (editable; displays as a blue dot if unread, or blank if read; clicking on the cell checks or unchecks an editable checkbox to mark the Wiki page as read or unread; standard column)
  * Category (standard column) 
  * Title (link to record; standard column)
  * ID (standard column)



  


Grouped by: 

  * Uncategorized at the top
  * Then alphabetically by Category



  


Sorted by: Title

  


Filters: 

  * Search for (plain text field; searches the Wiki page body memos; defaults to blank) 
  * Unread only (checkbox; defaults to unchecked; if checked, the report displays only pages that are unread for the current user) 



  


Buttons: 

  * New Page (standard button; opens a blank new Wiki page) 



  


Menu Visibility: 

  * Members Menu: All users (since all have access to this menu) 
  * Admin Menu: Super Admins & Regional Reps (by virtue of being on the Admin menu)



  


Other Notes: 

  * The Read/Unread checkbox saves automatically when edited.
