3.1. Project Management Incidents Report

  


Requirements

This is a report of Incidents with a Type that includes the Project Management section.

  


This is accessed from the "Project Management" menu option in the Incidents section of the Home Menu. 

  


Title: Project Management Incidents

  


Columns: 

  * Read (shows read/unread checkmark) 
  * Assignee
  * Title (link to incident) 
  * Type
  * Contact
  * Organization
  * Followup Date (editable)
  * Priority (editable)
  * Status (editable)
  * Start Date (editable)
  * Due Date (editable)
  * Finished Date (editable)
  * Status Summary (editable)
  * Workflow (editable)
  * Stage (editable)
  * ID



  


Grouped by: Selection in the "Group by" filter

  * If Incident Status: Doing, To Do, Done 
  * If Workflow: "N/A", then alphabetically 
  * If Due Status: Overdue, Coming Due (due within 7 days), Scheduled (due date further out than 7 days), Unscheduled (no due date) 



  


Sorted by: 

  * Earliest of Followup Date or Due Date, then
  * Priority, then
  * Title



  


Filters: 

  * Assignee (drop list of all active contacts with the Incident Assignee checkbox set; defaults to blank = all) 
  * Type (multi-select drop list of all active Incident Types; defaults to blank = all)
  * Workflow (single-select drop list of all active Incident Workflows; defaults to blank = all)
  * Stage (multi-select drop list of active Incident Stages, filtered to the selected Workflow; default to blank = all)
  * Tags (multi-select drop list of Incident Tags; default to blank = all) 
  * Open incidents only (checkbox; default is checked)
  * Group by (drop list of the following: 
    * Incident Status 
    * Workflow
    * Due Status (default grouping) 



  


Buttons: 

  * Save Changes



  


Menu Visibility: Visible for all users

  


Save Type (for editable reports): Save Button

  


Other Notes: 

  * N/A
