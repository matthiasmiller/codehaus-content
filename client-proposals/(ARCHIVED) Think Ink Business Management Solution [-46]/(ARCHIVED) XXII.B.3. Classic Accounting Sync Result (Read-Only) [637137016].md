22.2.3. Classic Accounting Sync Result (Read-Only)

  


Requirements

Overview: This is a record that will track syncing history between Classic Accounting and the Solution.

  


Accessed via: Classic Accounting Sync Results Report (see corresponding section of this proposal)

  


Sections and Fields: 

  * ID (read-only, auto-number) 
  * Date/Time (read-only; determined by log's activity history; format: mm/dd/yyyy h:mm AM)
  * Username (read-only; read-only user name of person requesting the manual sync)
  * Success (checkbox; auto-set and read-only; set via import if it succeeds)
  * Error Log (memo; auto-filled and read-only)
    * This will come directly from the script -- whatever it says went sideways. It will be a technical log.



  


Data Access: 

  * Visibility: Admin Only
  * Editability: Auto-generated



  


Other Notes:

  * The error log will be automatically generated by the sync process itself.
  * This will appear as a report alert to administrators.



  


Development Specification

Mockups: None needed; to be auto-generated
