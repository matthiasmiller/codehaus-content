5.4.8. Claim - Other Party Contact & Insurance Section

*Done. 

  


  * Other Party Contact & Insurance section: 
    * Other Party Contact (label for the following "Other Party" fields)
    * Name (optional; plain text; defaults to the contents of the "Other Driver" field, but can be overridden; note that editing "Other Driver" after this field has been set will reset this field to match)
    * Address 1 (optional; plain text)
    * Address 2 (optional; plain text)
    * City (optional; plain text)
    * State (no label; optional; drop list of state abbreviations)
    * Zip (no label; optional; plain text field for zip code)
    * Phone 1 (optional; plain text; auto-formats)
    * Phone 2 (optional; plain text; auto-formats)
    * Email (optional; plain text; validation error message if an invalid email address is entered: "Please enter a valid email address.")
    * Notes (optional; memo)



  


  * Other Party Insurance (label for the following "Other Party Insurance" fields)
  * Name (optional; drop list of active non-client type Organization Contacts; with the following details / behaviors: 
    * when a Name is selected, the following fields are auto-set: 
      * "Address 1"; 
      * "Address 2"; 
      * "City"; 
      * "State"; 
      * "Zip")
  * Contact (optional; plain text)
  * Address 1 (optional; plain text; is auto-set, but remains editable, when Other Party insurance "Name" is set above)
  * Address 2 (optional; plain text; is auto-set, but remains editable, when Other Party insurance "Name" is set above)
  * City (optional; plain text; is auto-set, but remains editable, when Other Party insurance "Name" is set above)
  * State (no label; optional; drop list of state abbreviations; is auto-set, but remains editable, when Other Party insurance "Name" is set above)
  * Zip (no label; optional; plain text field for zip code; is auto-set, but remains editable, when Other Party insurance "Name" is set above)
  * Phone 1 (optional; plain text; auto-formats)
  * Ext (optional; plain text; for documenting an extension number for "Phone 1")
  * Phone 2 (optional; plain text; auto-formats)
  * Ext (optional; plain text; for documenting an extension number for "Phone 2")
  * Fax (optional; plain text; auto-formats)
  * Email (optional; plain text; validation error message if an invalid email address is entered: "Please enter a valid email address.")
  * Notes (optional; memo)
